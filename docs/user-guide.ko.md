# ragkit 사용자 가이드

ragkit은 문서를 업로드하고 자연어로 검색할 수 있는 RAG 플랫폼입니다. 관리자 UI(http://localhost:5173)를 통해 모든 기능을 사용할 수 있습니다.

## 1. 로그인

관리자 UI에 접속하면 로그인 화면이 나타납니다. 관리자로부터 발급받은 이메일과 비밀번호를 입력하세요.

---

## 2. 서비스 선택

ragkit은 **서비스** 단위로 문서와 멤버를 분리합니다. 로그인 후 상단 헤더의 드롭다운에서 작업할 서비스를 선택하세요. 선택한 서비스는 브라우저에 저장되어 다음 접속 시에도 유지됩니다.

> 드롭다운에 서비스가 하나도 보이지 않는다면 superadmin에게 서비스 생성 또는 멤버 초대를 요청하세요.

---

## 3. 문서 업로드

**Documents** 페이지 상단의 Upload 폼을 사용합니다.

| 필드 | 설명 |
| ---- | ---- |
| Title | 문서를 식별할 이름 (필수) |
| File | 업로드할 파일 (필수) |
| Description | 문서에 대한 부가 설명 (선택) |

**Upload** 버튼을 누르면 파일 저장 → 텍스트 추출 → 임베딩 → 벡터 저장까지 순서대로 처리됩니다. 완료되면 우상단에 성공 알림이 표시됩니다.

> **업로드가 실패하는 경우**
> - 임베딩 서비스(Ollama)가 실행 중이지 않으면 오류 알림이 표시됩니다. Ollama를 먼저 실행한 뒤 다시 시도하세요.
> - 텍스트를 추출할 수 없는 파일(예: 스캔 이미지 PDF)은 업로드에 실패할 수 있습니다.

---

## 4. 문서 목록 및 상태 확인

업로드한 문서는 **Documents** 페이지 하단 목록에서 확인할 수 있습니다.

- 제목으로 필터링하려면 상단 검색창에 키워드를 입력하세요.
- **Status** 컬럼에서 인덱싱 상태를 확인할 수 있습니다. `indexed` 상태여야 검색에 사용됩니다.

---

## 5. 문서 상세 및 관리

목록에서 문서 제목을 클릭하면 상세 페이지로 이동합니다.

### 미리보기

**Preview** 버튼을 클릭하면 PDF 및 텍스트 파일을 브라우저에서 바로 확인할 수 있습니다.

### 다운로드

**Download** 버튼으로 원본 파일을 내려받을 수 있습니다.

### 파일 교체

새 파일로 교체하려면 **Replace file** 섹션에서 파일을 선택하고 **Replace and re-index** 버튼을 클릭하세요. 기존 벡터가 삭제되고 새 파일로 재인덱싱됩니다.

### 삭제

**Delete** 버튼을 클릭하면 문서와 관련된 모든 데이터(파일, 벡터)가 삭제됩니다. 삭제는 되돌릴 수 없습니다.

---

## 6. 검색

**Search** 페이지에서 자연어로 문서를 검색할 수 있습니다.

| 필드 | 설명 |
| ---- | ---- |
| Query | 검색할 내용을 자연어로 입력 (필수) |
| Top K | 반환할 최대 결과 수 (기본 5, 최대 50) |
| Document ID | 특정 문서 내에서만 검색하려면 해당 문서의 ID 입력 (선택) |

검색 결과는 관련도 점수(`score`)와 함께 청크 단위로 표시됩니다. 결과의 문서 제목을 클릭하면 해당 문서 상세 페이지로 이동합니다.

> `indexed` 상태인 문서만 검색 결과에 포함됩니다.

## 7. 멤버 관리 *(service admin 이상)*

사이드바의 **Members** 메뉴는 현재 서비스의 admin 역할을 가진 사용자에게만 표시됩니다.

### 멤버 추가

페이지 상단 폼에서 유저를 선택하고 역할을 지정한 뒤 **Add Member**를 클릭하세요. 이미 해당 서비스의 멤버인 유저는 목록에 표시되지 않습니다.

### 역할 변경

멤버 목록의 역할 드롭다운을 변경하면 즉시 반영됩니다.

| 역할 | 권한 |
| ---- | ---- |
| `admin` | 문서 읽기·쓰기·삭제, 멤버 관리 |
| `member` | 문서 읽기·쓰기·삭제 |
| `viewer` | 문서 읽기·검색만 가능 |

### 멤버 제거

**Remove** 버튼을 클릭하면 해당 유저가 서비스에서 제거됩니다.

---

## 8. 서비스 관리 *(superadmin 전용)*

사이드바의 **Services** 메뉴는 superadmin에게만 표시됩니다.

### 서비스 생성

Name(표시 이름)과 Slug(영문 고유 식별자)를 입력하고 **Create**를 클릭하세요.

### 서비스 삭제

목록에서 **Delete** 버튼을 클릭하세요. 서비스를 삭제해도 해당 서비스의 문서 데이터는 별도로 정리해야 합니다.

---

## 9. 문서 상태 설명

| 상태 | 의미 |
| ---- | ---- |
| `pending` | 파일 저장 완료, 인덱싱 대기 중 |
| `chunking` | 텍스트를 청크로 분할 중 |
| `embedding` | 임베딩(벡터 변환) 처리 중 |
| `indexed` | 인덱싱 완료, 검색 가능 |
| `failed` | 인덱싱 실패 (상세 페이지에서 오류 메시지 확인 가능) |

---

## 10. 지원 파일 형식

| 형식 | 확장자 |
| ---- | ------ |
| PDF | `.pdf` |
| Word | `.docx`, `.doc` |
| PowerPoint | `.pptx`, `.ppt` |
| Excel | `.xlsx`, `.xls` |
| 텍스트 | `.txt`, `.md`, `.markdown`, `.rst`, `.log` |

위 형식 외의 파일은 텍스트로 읽기를 시도하며, 읽을 수 없는 경우 업로드가 실패합니다.
